Drone ci for Gitea cicd工作流
持续集成 CI/CD Gitea
CI/CD
发布日期:   2019-10-14
文章字数:   790
阅读时长:   3 分
阅读次数:  
  • Gitea官网 https://docs.gitea.io
  • Gitea 是一个开源社区驱动的轻量级代码托管解决方案,他是从Gogs发展而来
  • Drone官网 https://docs.drone.io
  • Drone是一种基于容器技术的持续交付系统。Drone使用简单的YAML配置文件（docker-compose的超集）来定义和执行Docker容器中的Pipelines。Drone与流行的源代码管理系统无缝集成，包括GitHub，GitLab, Enterprise，Bitbucket,gogs,gitea等。

一、安装gitea

从gitea官网下载安装文件 https://docs.gitea.io 可以下载最新版本，我这里是1.9.2版本

1.从二进制安装
$ wget -O gitea https://dl.gitea.io/gitea/1.9.2/gitea-1.9.2-linux-amd64
$ chmod +x gitea
$ ./gitea web

启动gitea后，在浏览器打开连接: http://localhost:3000 进行配置。配置文件存在custom/conf/app.ini。

2.从docker安装
  • docker 直接启动安装
    首先你需要pull镜像：
    $ docker pull gitea/gitea:1.9.2
    如果要将git和其它数据持久化，你需要创建一个目录来作为数据存储的地方：
    $ sudo mkdir -p /var/lib/gitea
    然后就可以运行 docker 容器了，这很简单。 当然你需要定义端口数数据目录：
    $ docker run -d --name=gitea -p 10022:22 -p 10080:3000 -v /var/lib/gitea:/data gitea/gitea:1.9.2
    然后容器已经运行成功，在浏览器中访问 http://hostname:10080 就可以看到界面了。
    注意：目前端口改为非3000时，需要修改配置文件 LOCAL_ROOT_URL = http://localhost:3000/
  • docker-compose安装
    编辑docker-composer文件
    $ vim gitea.yml
version: "3.7"
networks:
gitea:
  external: false
services:
server:
  image: gitea/gitea:1.9.2
  environment:
    - USER_UID=1000
    - USER_GID=1000
  restart: always
  networks:
    - gitea
  volumes:
    - ./data:/data
  ports:
    - "3000:3000"
    - "222:22"
    启动、停止方式
    $ docker-compose -f gitea.yml up -d
$ docker-compose -f gitea.yml down
    然后容器已经运行成功，在浏览器中访问 http://hostname:3000 就可以看到界面了。

二、安装Drone

编辑docker-compose文件，’vim drone.yml’

version: "3.7"

services:
  drone-server:
    image: drone/drone:1.4
    container_name: drone_server
    ports:
    - "8000:80"
    - "8443:443"
    environment:
      - DRONE_OPEN=true
      - DRONE_GITEA=true
      - DRONE_DEBUG=true
      - DRONE_PROVIDER=gitea
      - DRONE_RUNNER_CAPACITY=2
      - DRONE_GIT_ALWAYS_AUTH=true
      - DRONE_SERVER_HOST=drone.company.com #drone的访问地址
      - DRONE_AGENTS_ENABLED=true
      - DRONE_GITEA_SERVER=https://try.gitea.io  #gitea的访问地址
      - DRONE_GITEA_CLIENT_ID=6522e99d-1adb-4ac1-bf79-e1b9bf3b9974  #gitea的客户端ID
      - DRONE_GITEA_CLIENT_SECRET=WF1o3cgAWQ2Nz7JgAma-1M-qC5EuNyXubtw7AaTZk90=  #gitea的客户端密钥
      - DRONE_GITEA_SKIP_VERIFY=false
      - DRONE_SERVER_PROTO=http
      - DRONE_TLS_AUTOCERT=false
      - DRONE_GIT_USERNAME=test  #一般为gitea账号
      - DRONE_GIT_PASSWORD=123456 
      - DRONE_RPC_SECRET=37a18ef42683494a67de20c87ac2d773
      - DRONE_LOGS_DEBUG=true
      - DRONE_DATABASE_DRIVER=mysql #mysql数据库连接方式(两者选其一)
      - DRONE_DATABASE_DATASOURCE=gitea:password@tcp(localhost:3306)/drone 
      #- DRONE_DATABASE_DRIVER=mysql #sqlite数据库
      #- DRONE_DATABASE_DATASOURCE=/data/database.sqlite 
      - DRONE_USER_CREATE=username:tfzxyinhao,xiaolu,dmin:true #设置drone管理员账号
    restart: always
    volumes:
    - /var/run/docker.sock:/var/run/docker.sock
    - /var/lib/drone_1.4:/data

  drone-agent:
    image: drone/agent:1.4
    container_name: drone_agent
    ports:
    - "3000:3000"
    depends_on:
    - drone-server
    restart: always
    environment:
      - DRONE_RPC_PROTO=http
      - DRONE_RPC_HOST=drone.company.com
      - DRONE_RPC_SECRET=37a18ef42683494a67de20c87ac2d773
      - DRONE_RUNNER_CAPACITY=2
      - DRONE_LOGS_DEBUG=true
      - DRONE_DEBUG=true
      - DRONE_LOGS_PRETTY=true
      - DRONE_LOGS_NOCOLOR=false
    volumes:
    - /var/run/docker.sock:/var/run/docker.sock

启动、停止drone

$ docker-compose -f drone.yml up -d
$ docker-compose -f drone.yml down


然后容器已经运行成功，在浏览器中访问 http://hostname:8000 就可以看到界面了。
通过gitea认证登录进去，会自动同步仓库
