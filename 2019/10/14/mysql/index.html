Mysql常用命令
Mysql
Mysql
发布日期:   2019-10-14
文章字数:   1k
阅读时长:   4 分
阅读次数:  

1.创建用户

CREATE USER 'username'@'host' IDENTIFIED BY 'password';

username:用户名；host：指定在哪个主机上可以登录，本机可用localhost，%通配所有远程主机；password：用户登录密码；

2.授权

grant all privileges on *.* to root@'%' IDENTIFIED BY "password";  
FLUSH PRIVILEGES;

格式：grant 权限 on 数据库名.表名 to 用户@登录主机 identified by “用户密码”；.代表所有权；@ 后面是访问MySQL的客户端IP地址（或是 主机名） % 代表任意的客户端，如果填写 localhost 为本地访问（那此用户就不能远程访问该mysql数据库了）。

3.基础命令

创建数据库：create database testdatabases;
删除数据库 ：drop database testdatabases;
查看数据库：show databases;
打开数据库：use databases;
查看表：show tables;
查看当前所在库：select database();
查看数据库版本：select version();
查看数据库所有用户：select user,host from mysql.user;
删除用户：drop user test@localhost ;
授予用户“test”通过外网IP对于该数据库“testdb”中表的创建、修改、删除权限,以及表数据的增删查改权限：
grant create,alter,drop,select,insert,update,delete on testdb.* to test@'%';

4.表插入数据

插入一个name字段：insert into table (`name`) values('55');
插入一个字段id： insert into table (`id`) values(3);
插入一行：insert into table values(1,'gz');
更新某行（修改）：update table set id=5 where name='55';
删除制定行：delete from table where name='55';

命令：insert into <表名> [( <字段名1>[,..<字段名n > ])] values ( 值1 )[, ( 值n )]
注意：insert into每次只能向表中插入一条记录。

5.查询表中的数据

查看表中所有数据：select * from table;
查看表中前2行数据：select * from table order by id limit 0,2;
删除表中编号为1的记录：delete from table where id=1;

6.更改数据库用户密码

5.7版本下的mysql数据库下已经没有password这个字段了，password字段改成了authentication_string

终端命令：mysqladmin -u root password “密码”
数据库命令：update user set password=password('123456') where user='root';
5.7版本数据库：update user set authentication_string=password('123456') where user='root';

7.备份数据库命令

导入数据库sql文件：mysql -u用户名 -p 数据库名 < 导入文件名.sql
导出数据库：mysqldump -u用户名 -p密码 数据库名 > 数据库名.sql
MySql的备份数据库命令：mysqldump -h 主机 -u数据户用户名 -p数据库密码 数据库 > bak.sql
MySql还原恢复数据库命令：mysql -h 主机 -u数据户用户名 -p数据库密码 数据库 < bak.sql
按日期导出数据库到指定文件夹：30 5 * * * mysqldump -uroot -p password database>/data/backup`date "+%Y%m%d"`.sql

8.远程操作数据库

格式：mysqldump -h主机名 -P端口 -u用户名 -p密码 –database 数据库名 > 文件名.sql

例如：mysqldump -h 192.168.1.100 -p 3306 -uroot -ppassword --database cmdb > /data/backup/cmdb.sql

导出的数据有可能比较大，不好备份到远程，这时候就需要进行压缩
格式：mysqldump -h主机名 -P端口 -u用户名 -p密码 –database 数据库名 | gzip > 文件名.sql.gz

例如：mysqldump -h192.168.1.100 -p 3306 -uroot -ppassword --database cmdb | gzip > /data/backup/cmdb.sql.gz

备份数据库结构，不备份数据
格式：mysqldump -h主机名 -P端口 -u用户名 -p密码 –no-data 数据库名1 数据库名2 数据库名3 > 文件名.sql

例如：mysqldump --no-data –databases db1 db2 cmdb > /data/backup/structure.sql

备份实例上所有的数据库
格式：mysqldump -h主机名 -P端口 -u用户名 -p密码 –all-databases > 文件名.sql

例如：mysqldump -h192.168.1.100 -p3306 -uroot -ppassword --all-databases > /data/backup/all_db.sql

备份同个库多个表
格式：mysqldump -h主机名 -P端口 -u用户名 -p密码 –database 数据库名 表1 表2 …. > 文件名.sql

例如：mysqldump -h192.168.1.100 -p3306 -uroot -ppassword cmdb t1 t2 > /data/backup/cmdb_t1_t2.sql
